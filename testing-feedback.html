Game: Testing & Feedback

Throughout development, we tested the game to ensure it functioned correctly. During a class showcase, we received feedback from peers and instructors, which is now being used to improve the experience.

A key point was that Granite should be more deeply integrated into gameplay, influencing player decisions rather than acting only as a coding assistant. Ideas included requiring a Granite-generated password to start the spaceship, using commands like “Start the engine”, or having it help plan routes between planets.

Blog: Testing & Feedback

We added a starry background to strengthen the space theme, but feedback showed it was too busy and distracting due to large stars. This was replaced with a subtler design using smaller stars to improve readability and atmosphere.

We also created a rocket animation to improve engagement, but testing revealed incorrect positioning and rotation, which broke immersion. Feedback highlighted that the rocket should align with the path direction.

To fix this, the JavaScript animation logic and SVG path were reworked. We tested different approaches using path length and scroll-based positioning, ultimately improving accuracy so the rocket follows the curve more naturally.
